# Глава 2.7. Веб-приложение

Веб-приложение — практическая часть работы: оно делает все предыдущие этапы (кластеризацию,
метакластеры, рекомендации, объяснения) доступными через интерактивный интерфейс.

## Технологии

- **Backend:** FastAPI (`src/web/backend/`), раздаёт статику и предоставляет JSON-API.
- **Frontend:** ванильный JavaScript + **Deck.gl** (`src/web/static/`) для
  высокопроизводительной отрисовки десятков тысяч точек.
- **Визуализация карты:** ортографическая проекция Deck.gl со слоями `ScatterplotLayer` (статьи) и
  `TextLayer` (подписи метакластеров, тем, статей).

## API backend

| Endpoint | Назначение |
| --- | --- |
| `GET /` | HTML-страница приложения |
| `GET /api/map/state` | Данные карты: точки статей, центры и подписи fine-тем и метакластеров |
| `GET /api/papers/{paper_id}` | Детали статьи: заголовок, аннотация, год, авторы, тема, метакластер, ссылка OpenAlex |
| `GET /api/clusters/info/{node_id}` | Сводка по теме/метакластеру: метка, размер, топ-термины, представительные статьи |
| `GET /api/authors/search?q=` | Автодополнение поиска авторов |
| `POST /api/recommendations/{author_id}` | Запуск пятиступенчатого pipeline рекомендаций |

Маршруты разнесены по модулям (`api_clusters.py`, `api_authors.py`, `api_recommendations.py`),
тяжёлое состояние карты собирается один раз при старте (`state.py`).

## Карта научного пространства

Главный экран — карта, на которой каждая точка это статья, окрашенная по своей fine-теме (оттенок
наследуется от метакластера, см. главу 2.4).

Навигация построена на **зуме**, а не на «проваливании» внутрь кластеров:

- **дальний зум** — видны крупные подписи **метакластеров** (40 крупных направлений);
- **средний зум** — проявляются подписи **fine-тем**;
- **глубокий зум** — появляются **заголовки отдельных статей**.

Подписи плавно проявляются/затухают по уровню зума (opacity fading), а пересечения подписей
устраняются встроенной коллизионной фильтрацией Deck.gl — карта остаётся читаемой на любом масштабе.

### Взаимодействие

- **Клик по статье** открывает боковую панель с деталями: заголовок, аннотация, год, авторы,
  fine-тема, метакластер, ссылка на OpenAlex.
- **Клик по подписи кластера** показывает временную всплывающую сводку (метка, размер, топ-термины,
  представительные статьи) — без смены уровня зума.

## Поиск авторов

Поле поиска с **автодополнением**:

- регистронезависимый поиск по подстроке (имя/фамилия);
- базовая транслитерация кириллицы в латиницу (можно искать русскими буквами);
- результаты ранжируются по числу публикаций;
- для каждого автора показываются имя, число статей, ссылка OpenAlex и последние работы.

Выбор автора запускает построение рекомендаций.

## Окно рекомендаций

После выбора автора выполняется pipeline из [главы 2.6](05_llm_reranking.md), и в модальном окне
показываются:

- предложенные авторы-кандидаты со ссылками на их профили OpenAlex;
- **тип рекомендации** для каждого (прямое тематическое / методологическое совпадение /
  взаимодополняющая экспертиза);
- краткое объяснение;
- компактные доказательства — попарные связи статей с короткой причиной и ссылками OpenAlex.

## Скриншоты для курсовой

Эти иллюстрации нужно подготовить (снять с работающего приложения, запустив
`uv run python scripts/run_app.py`):

1. **Общий вид карты** научного пространства (дальний зум, видны метакластеры).
2. **Средний зум** — проявившиеся подписи fine-тем.
3. **Глубокий зум** — точки статей и их заголовки, цветовое кодирование тем.
4. **Боковая панель статьи** — детали выбранной публикации.
5. **Всплывающая сводка кластера** — топ-термины и представительные статьи.
6. **Поиск автора** — автодополнение с подсказками.
7. **Окно рекомендаций** — кандидаты, типы связей, объяснения и доказательства.

Рекомендуется вставлять их в главы 2.7 и 3.5 (демонстрация интерфейса).

## Результат этапа

Работающее веб-приложение, интегрирующее все подсистемы: запуск через
`uv run python scripts/run_app.py` (параметры берутся из `configs/app.yaml`).
